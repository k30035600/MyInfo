# 배포 진행 체크리스트

## 1. GitHub에 최신 코드 올리기 (필수)

Railway는 GitHub 저장소를 기준으로 배포하므로, 먼저 변경 사항을 push 하세요.

```powershell
cd d:\OneDrive\Cursor_AI_Project\MyInfo
git add .
git status
git commit -m "chore: Railway 배포 설정 (start_web.sh, PORT 기본값, 문서)"
git push origin main
```

- 이미 Railway에 이 저장소를 연결해 두었다면 → **push 만 하면 자동 재배포** 됩니다.
- 아직 연결 안 했다면 → 아래 2단계로 새로 배포하세요.

---

## 2. Railway에서 배포하기

### A) 이미 연결된 프로젝트가 있는 경우

- `main` 에 push 하면 **자동 재배포** 됨.
- 대시보드: **https://railway.app** → 해당 프로젝트 → **Deployments** 에서 진행 상황 확인.

### B) 처음 배포하는 경우

1. **https://railway.com/new** 접속
2. **Deploy from GitHub repo** 선택
3. **MyInfo** (또는 사용 중인 저장소) 선택
4. 빌드 완료 후 **서비스** 클릭 → **Settings** → **Networking** → **Generate Domain**
5. 생성된 URL로 접속해 동작 확인

---

## 3. 배포 후 확인

- **Deployments** 탭에서 **Build Logs** / **Deploy Logs** 로 에러 여부 확인
- **Generate Domain** 으로 만든 URL 로 접속: `https://xxx.up.railway.app`
- 문제 있으면 `README_Railway.md` 의 "배포가 중단되었을 때" 참고

---

**요약:** `git add` → `git commit` → `git push origin main` 한 뒤, Railway 대시보드에서 자동 배포 확인 또는 **railway.com/new** 로 새 배포.
