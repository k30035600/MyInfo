📋 금융거래 고급분석

시작하기

금융정보통합정보(MyCash)는 은행 및 신용카드의 거래 내역을 하나로 모아 통합 관리하는 시스템입니다. 전처리 → 카테고리 분류 → 분석 순으로 사용합니다.

서버 실행

프로젝트 루트에서 start-server.bat을 실행하거나, 터미널에서 python app.py를 실행한 뒤 브라우저로 접속합니다.

cd MyInfo 경로 python app.py

실행 시 패키지 설치(pip install) 후 Waitress WSGI 서버가 뜨며, 기본 주소는 http://localhost:5000입니다. 금융정보 전처리는 /cash/, 카테고리는 /cash/category, 기본분석은 /cash/analysis/basic입니다.

폴더 구조

금융정보 관련 파일은 모두 MyCash 폴더 안에서 다룹니다.

  • Source: 각 은행에서 내려받은 거래 내역 원본(.xls, .xlsx)을 넣는 폴더입니다. 파일명에 국민은행, 신한은행, 하나은행이 포함되어야 인식합니다.
  • bank_before.xlsx: 전처리 결과가 저장되는 파일입니다. Source 파일들을 통합·표준화한 결과입니다.
  • bank_category.xlsx: 카테고리 분류 규칙(분류, 키워드, 카테고리)이 저장되는 파일입니다. 자동 생성되며, 수동 수정도 가능합니다.
  • bank_after.xlsx: 카테고리를 적용한 최종 거래 데이터입니다. 기본분석은 이 파일을 사용합니다.

지원 은행 및 파일 형식

Source 폴더에는 .xls 또는 .xlsx 파일만 사용합니다. HTML 등 다른 형식은 지원하지 않습니다.

  • 국민은행: .xls, .xlsx 모두 지원합니다.
  • 신한은행: .xls, .xlsx 모두 지원합니다.
  • 하나은행: .xls, .xlsx 모두 지원합니다.

파일명 예: 국민은행_285102-04-255016_2024.xlsx, 신한은행_110-478-945020_2024.xls, 하나은행_433-910335-80107_2024.xls

주요 기능

📊 전처리

Source의 .xls/.xlsx를 읽어 표준 컬럼(거래일, 거래시간, 은행명, 계좌번호, 입금액, 출금액, 잔액, 구분, 적요, 내용, 거래점, 송금메모, 메모, 카테고리)으로 통합합니다. bank_before.xlsx로 저장됩니다.

🏷️ 카테고리

분류(전처리/후처리/거래방법/거래지점/기타거래)와 키워드 기반 자동 분류를 지원합니다. bank_category.xlsx를 수정한 뒤 "카테고리 적용"으로 bank_after.xlsx를 갱신합니다.

📈 기본분석

전체 통계, 적요별·월별·은행/계좌별 요약, 차트를 통해 거래 패턴을 빠르게 파악합니다.

카테고리 체계

카테고리는 분류와 키워드 → 카테고리 매핑으로 구성됩니다.

  • 전처리: 은행명·기관 약어(예: NH→농협, KB→국민, 주금공→주택금융공사) 정리용입니다.
  • 후처리: 괄호·공백 등 문자열 보정(예: ((→(, ))→), []→space)에 사용합니다.
  • 거래방법: ATM, CMS, 뱅킹, 스마트, 인터넷, 지로, 카드, 현금, 대출, 취소, 기타 등입니다. 적요/내용/거래점/송금메모에서 키워드를 추출해 자동 매칭됩니다.
  • 거래지점: 은행명(거래점) 형식으로 거래점별 카테고리를 둡니다. 괄호 균형·숫자 제거 등 정규화가 적용됩니다.
  • 기타거래: 위 분류에 포함되지 않는 항목을 정리할 때 사용합니다.

1차(입출금) → 2차(전처리) → 3차(거래방법) → 5차(거래지점) → 6차(기타거래) 순으로 적용됩니다. "카테고리 생성"은 bank_before 기준으로 규칙을 자동 생성하고, "카테고리 적용"은 그 규칙으로 bank_after를 만듭니다.

페이지별 사용법

은행거래 전처리 (/bank/)

  • MyBank → Source 폴더에 국민/신한/하나 은행 거래 내역 .xls 또는 .xlsx를 넣습니다.
  • 전처리 페이지에서 "전처리 실행"(또는 동일 기능 버튼)을 클릭합니다. 서버가 Source를 읽어 통합하고, bank_before.xlsx·bank_category.xlsx·bank_after.xlsx를 생성·갱신합니다.
  • 전처리 전/후 데이터를 테이블로 비교할 수 있습니다. 은행, 년·월 등 필터로 범위를 줄여 확인하세요.
  • 데이터가 없으면 "Source에 .xls/.xlsx가 있는지", "파일명에 은행명이 포함되는지"를 확인합니다.

은행거래 카테고리 (/bank/category)

  • 카테고리 테이블: 분류, 키워드, 카테고리 컬럼으로 구성됩니다. 행 추가·삭제·수정으로 규칙을 관리합니다.
  • "카테고리 생성": bank_before.xlsx를 기준으로 전처리/후처리/거래방법/거래지점 규칙을 자동 생성합니다. 기존 bank_category.xlsx가 있으면 덮어쓸 수 있으므로, 필요 시 백업 후 실행하세요.
  • "카테고리 적용": 현재 카테고리 테이블 규칙을 bank_before에 적용해 bank_after.xlsx를 생성·갱신합니다. 적용 후 "카테고리 적용후" 섹션에서 데이터 미리보기로 결과를 확인할 수 있습니다.
  • 은행 필터·📄 출력: 페이지 헤더의 "은행명" 드롭다운으로 특정 은행만 보거나, "📄 출력" 버튼으로 현재 화면을 인쇄·PDF 저장할 수 있습니다. 출력 시 제목은 "은행거래 카테고리 적용후 (bank_after.xlsx)"로 표시됩니다.
  • 카테고리별 통계, 월별 추이 등도 이 페이지에서 확인할 수 있습니다.

은행거래 기본분석 (/bank/analysis/basic)

  • 상단 통계: 거래건수, 입금건수, 출금건수, 입금합계, 출금합계, 순잔액을 한눈에 확인합니다.
  • 입/출금 내역: 적요별로 입금·출금·차액을 표시합니다. 행을 클릭하면 해당 적요의 상세 거래 내역이 우측에 나타납니다.
  • 입금/출금/차액 그래프: 적요별 입금·출금·차액을 막대 차트로 비교합니다. x축은 6자까지 잘려 표시됩니다.
  • 은행별/계좌별 통계: 은행·계좌 단위 입금·출금·순잔액 집계와 파이 차트를 제공합니다. 합계 행은 테이블 하단에 고정됩니다.
  • 은행 필터: 페이지 헤더의 "은행명" 드롭다운으로 특정 은행만 분석할 수 있습니다.
  • 📄 출력: 헤더의 "📄 출력" 버튼을 누르면 브라우저 인쇄 대화상자가 열립니다. 현재 필터가 적용된 화면 그대로 인쇄·PDF 저장할 수 있습니다.

주요 데이터 파일·컬럼

bank_before.xlsx (전처리 결과)

Source 파일들을 통합한 표준 컬럼: 거래일, 거래시간, 은행명, 계좌번호, 입금액, 출금액, 잔액, 구분, 적요, 내용, 거래점, 송금메모, 메모, 카테고리(비어 있을 수 있음) 등.

bank_after.xlsx (카테고리 적용 후)

bank_before와 동일한 구조에, 카테고리 규칙을 적용한 카테고리 컬럼이 채워진 최종 데이터입니다. 거래일(yy/mm/dd), 적요, 거래방법, 입출금, 거래점, 송금메모, 입금액, 출금액, 구분 등이 기본분석·출력에 사용됩니다.

문제 해결

데이터가 표시되지 않음

  • Source 폴더에 .xls 또는 .xlsx 파일이 있는지 확인하세요. 파일명에 국민은행/신한은행/하나은행이 들어가야 합니다.
  • 전처리를 실행한 뒤 bank_before.xlsx가 MyBank 폴더에 생성되었는지 확인하세요. 파일이 비어 있거나 Excel 등에서 열려 있으면 읽기 실패할 수 있습니다.
  • 브라우저 개발자 도구(F12) → Console 탭에서 오류 메시지를 확인하세요. API 오류 시 서버 터미널 로그도 함께 확인합니다.

카테고리·bank_after가 생성/적용되지 않음

  • bank_category.xlsx가 MyBank 폴더에 있는지 확인하세요. 없으면 카테고리 페이지에서 "카테고리 생성"을 먼저 실행합니다. 전처리만 하고 카테고리 생성을 하지 않았을 때 자주 발생합니다.
  • "카테고리 적용" 전에 카테고리 테이블에 필요한 규칙(분류, 키워드, 카테고리)이 들어 있는지 확인하세요.
  • bank_after.xlsx를 Excel 등에서 열어둔 상태면 쓰기 오류가 날 수 있습니다. 파일을 닫은 뒤 다시 "카테고리 적용"을 실행하세요.

전처리·통합 오류

  • 원본 파일이 해당 은행 표준 형식(날짜, 적요, 입금, 출금 등 필수 컬럼 포함)인지 확인하세요. 수동 편집·다운로드 오류로 형식이 깨진 경우 제외될 수 있습니다.
  • 국민은행, 신한은행, 하나은행 모두 .xls, .xlsx 형식을 지원합니다.
  • 인코딩은 UTF-8 또는 CP949를 사용합니다. 한글이 깨지면 저장 시 인코딩을 바꿔 다시 시도해 보세요.

서버 연결·실행 오류

  • 서버가 실행 중인지 확인하세요. python app.py 또는 start-server.bat으로 띄웁니다.
  • 포트 5000이 다른 프로그램에서 사용 중이면 연결이 안 될 수 있습니다. 해당 프로그램을 종료하거나, app.py에서 port 값을 변경한 뒤 다시 실행하세요.
  • 방화벽에서 localhost:5000 접속이 차단되지 않았는지 확인하세요.

접속 주소

  • 홈페이지: http://localhost:5000
  • 금융거래 도움말: http://localhost:5000/help
  • 은행거래 전처리: http://localhost:5000/bank/
  • 은행거래 카테고리: http://localhost:5000/bank/category
  • 은행거래 기본분석: http://localhost:5000/bank/analysis/basic
  • 은행거래 도움말: http://localhost:5000/bank/help
