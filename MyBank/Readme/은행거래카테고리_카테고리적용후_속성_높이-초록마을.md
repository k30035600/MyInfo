# 은행거래 카테고리 - 카테고리적용후 속성 및 높이

은행거래 카테고리 페이지(`/bank/category`)에서 **카테고리적용후** 영역의 구성요소와 높이·패딩·오버플로우 관련 속성을 정리한 문서입니다.

---

## 1. HTML 구조 (구성요소)

```
.chart-card                          ← 카테고리적용후 카드 (인라인: margin-top 0, margin-bottom 20px)
├── div (헤더 행, flex)
│   ├── h2 "카테고리 적용후"
│   └── span#category-applied-filename  ("파일명: bank_after.xlsx")
└── .bank_category-container         ← 스크롤 영역 (인라인으로 높이·오버플로우 지정)
    └── #category-applied-content     ← 테이블 담는 div (인라인: width/height 100%, overflow-y auto)
        └── #category-applied-table / #category-applied-table-index
```

- **파일**: `MYBCBANK/templates/category.html`
- **테이블 ID**: 데이터 있을 때 `#category-applied-table`, 인덱스 모드 시 `#category-applied-table-index`
- **필터 드롭다운**: `#filter-category-applied-${col}` 또는 `#filter-category-applied-index-${col}`

---

## 2. 높이·패딩·오버플로우 속성 요약

| 속성 | 값 | 비고 |

| margin-top | 0 | 인라인 |
| margin-bottom | 20px | 인라인 |
| display | flex | 인라인 |
| flex-direction | column | 인라인 |
| **공통 .chart-card** | | |
| background | white | |

### 2.2 헤더 행 (제목 + 파일명)

| 요소 | 속성 | 값 | 비고 |

| | align-items | center | |

| | gap | 10px | |

### 2.3 테이블 스크롤 영역 (.bank_category-container, 카테고리적용후용)

**이 블록에만 적용되는 인라인 스타일** (category.html 466행):

| 속성 | 값 | 비고 |
|------|-----|------|
| overflow-x | auto | 인라인 |
| overflow-y | auto | 인라인 |
| height | 600px | 인라인 → 미디어쿼리보다 우선 |
| max-height | 600px | 인라인 |
| min-height | 600px | 인라인 |
| flex | none | 인라인 |

**공통 .bank_category-container** (인라인이 없는 경우/다른 블록):

| 속성 | 값 | 비고 |
|------|-----|------|
| flex | 1 | |
| overflow-y | auto | |
| overflow-x | hidden | |
| min-height | 0 | |
| **768px 이하** | height | 300px | 카테고리적용후는 인라인 600px로 인해 미적용 가능 |
| **480px 이하** | height | 250px | 동일 |

- 인라인이 지정된 카테고리적용후 영역은 **뷰포트가 768px/480px 이하여도 높이 600px이 유지**됩니다. 반응형으로 줄이려면 인라인 height를 제거하고 클래스만 두거나, 미디어쿼리에서 `!important` 또는 더 구체적인 선택자로 덮어써야 합니다.

## 3. 반응형(미디어 쿼리) 요약

- **max-width: 768px**
  - .chart-card: padding 15px
  - .chart-card h2: font-size 16px
  - .bank_category-container: height 300px *(단, 카테고리적용후는 인라인 600px로 덮일 수 있음)*
  - table th, td: padding 6px 4px, font-size 12px

- **max-width: 480px**
  - .chart-card: padding 10px
  - .chart-card h2: font-size 14px
  - .bank_category-container: height 250px *(동일하게 인라인 우선 가능)*
  - table th, td: padding 5px 3px, font-size 11px

- **max-width: 360px**
  - .chart-card h2: font-size 12px
  - table th, td: padding 4px 2px, font-size 10px

---

## 4. 요약 한 줄

- **카테고리적용후 카드**: `.chart-card` + 인라인 `margin-top:0; margin-bottom:20px; flex column`
- **스크롤 영역**: `.bank_category-container`에 인라인 `height:600px; max-height:600px; min-height:600px; flex:none; overflow-x/y:auto`
- **테이블 영역**: `#category-applied-content` 인라인 `width:100%; height:100%; overflow-y:auto`
- **테이블**: `#category-applied-table` 등 `font-size:11px`, thead `sticky; top:0; background:#f5f5f5; z-index:10`
- **높이 제어**: 카테고리적용후 높이는 인라인 600px가 우선하므로, 반응형에서 줄이려면 인라인 제거 또는 미디어쿼리에서 별도 덮어쓰기 필요
