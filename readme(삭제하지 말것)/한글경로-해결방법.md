# 한글 깨짐 확실한 원인 제거

## 왜 오류가 나나요?

- Cursor가 명령을 실행할 때 **임시 PowerShell 스크립트**를 만듭니다.
- 이 스크립트 안에 **현재 워크스페이스 경로**가 들어갑니다.  
  예: `C:\Users\k3003.삼아개발\OneDrive\Cursor`
- 경로에 **한글(삼아개발)** 이 있으면, 스크립트를 저장하는 과정에서 인코딩이 깨져  
  `?아개발`, `ư` 같은 문자로 바뀝니다.
- 그 결과:
  - `Set-Location : '...' 경로를 찾을 수 없습니다`
  - `New-Item : 잘못된 문자가 포함되어 있습니다`

**즉, 원인 = 워크스페이스 경로에 한글이 있고, Cursor가 그 경로를 임시 스크립트에 넣을 때 깨지는 것.**

## 확실한 해결 (한 번만 하면 됨)

Cursor가 **한글이 없는 경로**만 쓰게 만들면 됩니다.  
실제 폴더를 옮기지 않고, **연결(Junction)** 만 만들어서 `C:\CursorWorkspace`로 접근하면 됩니다.

### 1단계: Junction 만들기

1. Cursor **통합 터미널**을 엽니다 (`` Ctrl+` ``).
2. 현재 폴더가 이 프로젝트 루트인지 확인한 뒤 아래를 실행합니다.

   ```powershell
   .\create-workspace-junction.ps1
   ```

3. "연결을 만들었습니다" 메시지가 나오면 성공입니다.  
   (`C:\CursorWorkspace`에 이미 일반 폴더가 있으면 오류가 납니다. 그 폴더를 옮긴 뒤 다시 실행하세요.)

### 2단계: Cursor에서 한글이 없는 경로로 열기

1. **파일** → **폴더 열기** (또는 **File** → **Open Folder**)
2. 경로에 **`C:\CursorWorkspace`** 입력 후 열기
3. 기존에 열었던 한글 경로 폴더는 닫아도 됩니다.

이제 Cursor의 워크스페이스 경로가 `C:\CursorWorkspace`가 되므로,  
임시 스크립트에 깨진 한글이 들어갈 **원인이 제거**됩니다.  
(파일은 그대로 OneDrive\Cursor에 있고, Junction은 그 폴더를 가리키는 연결일 뿐입니다.)

## 요약

| 구분 | 내용 |
|------|------|
| **원인** | 워크스페이스 경로에 한글 → Cursor 임시 스크립트 인코딩 깨짐 |
| **확실한 원인 제거** | `create-workspace-junction.ps1` 실행 후, Cursor에서 **C:\CursorWorkspace** 로만 열기 |

이후에는 같은 프로젝트를 열 때 **항상 C:\CursorWorkspace** 로 열면 한글 경로 오류가 나지 않습니다.
